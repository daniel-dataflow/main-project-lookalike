# 통합 알림 및 자동 복구 시스템 관제 명세서 (Alert & Auto-Recovery)

> **문서 대상**: 코어 인프라 백엔드 개발자 및 서버 시스템 관리자(Sysops)  
> **관련 핵심 컴포넌트 목록**:
> - `services/slack_notifier.py` — 메인 Slack 훅 알림망 발송 핵심 로직
> - `services/auto_recovery.py` — 인프라 타겟 자동 복구망 연동 체계

---

## 1. 관제 시스템 개요 및 목적

운영 중인 서비스의 로그 모니터링 시스템에서 파생되는 핵심 인시던트 이벤트(에러, 장애 복구 등)를 Slack Incoming Webhook 체인을 통해 외부 채널에 실시간으로 브로드캐스트 전파 수행함. 알림의 도배 스팸을 막기 위해 **유예 기간, Circuit Breaker, Rate Limiter** 등 강력한 통제 방어망을 채택함.

| 알림 전파 유형 | 발동 감지 트리거 조건 | 출력 이모지 |
|------|--------|--------|
| **개별 치명 에러 탐지망** | 인프라 내 CRITICAL 혹은 ERROR 로그 문자열 감지 적발 시 즉결 타전 | 🚨 / ❌ |
| **순간 에러 누적 폭주망(Spike)** | 지정된 스팬 타임박스 윈도우 내 에러 볼륨 임계선 돌파 시 병합 타전 | ⚠️ |
| **자동 힐링 복구 처리망** | 데드락 오작동 컨테이너 자동 킬 재시작 구제 후 결과 리포트 | 🔄 / ⚠️ |
| **방어용 차단기(Circuit) 발동** | 서버 도배 마비 감지로 자동 차단(Open) 진입 직전 발송하는 최후의 경고 | 🚫 |

### 핵심 시스템 아키텍처 개선 적용안
- **`.env` 환경변수 록킹 통합화**: UI 제어로 휘발될 수 있는 시스템 영속 설정들을 파일(Environment Variable)로 완전 이관 및 하드 백업화 달성.
- **서버 기동 노이즈 알람 유예(Grace) 기간 적용**: 서버 전체 재부팅 콜드 스타트 후 최초 15분간 에러성 알림 발송 임의 강제 억제 뮤트화 (Airflow/Hadoop 등 초기화 도배 스팸 방어).
- **부팅 노이즈 영구 박제 필터**: 널리 알려진 초기화 에러 텍스트 패턴은 유예 시간 이후로도 100% 영구 무시 드롭 패스.

## 2. 시스템 환경 변수 스키마 세부 설정 (`.env`)

서버 런타임이 파괴 및 재시작된 이후에도 불변 유지 보존되는 시스템 디폴트 기본값을 명시함. 어드민 UI 저장분은 인메모리로 일시 핫반영되나, 데몬 재시작 시 하기 세팅값으로 무조건 롤백 초기화됨.

```dotenv
# === Slack 알림 시스템 콘피그 랩퍼 ===
# 지정 Incoming Webhook 외부 통신 URL (필수)
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/T.../B.../xxx

# 최소 반응 위협 레벨선: CRITICAL | ERROR | WARN 중 택일 (기본값: CRITICAL)
SLACK_MIN_ALERT_LEVEL=CRITICAL

# 알림 타전 허용 시간대 바운더리 KST 0~23 입력 (빈칸 시 24시간 풀가동)
SLACK_ACTIVE_HOURS_START=
SLACK_ACTIVE_HOURS_END=

# 에러 단기 급증 감별 스파이크(Spike) 제어 트리거
SLACK_SPIKE_THRESHOLD=15             # 윈도우 허용 상한 카운트
SLACK_SPIKE_WINDOW_SEC=600           # 감지를 위한 타임박스 윈도우 시간 (초)

# 어뷰징 도배 방어 체계 - Rate Limiter (예: 5분 내 최대 송출 N건 제한)
SLACK_MAX_ALERTS_PER_WINDOW=10
SLACK_RATE_WINDOW_SEC=300

# 어뷰징 도배 방어 체계 - Circuit Breaker 통신 차단 오픈 지속 유예 시간 (초)
SLACK_CIRCUIT_OPEN_SEC=1800

# 기동 인입 통제 유예 기간: 컨테이너 떼부팅 시작 15분(900초)간 알림 전송 원천 억제
SLACK_STARTUP_GRACE_SEC=900
```

## 3. 알람 파이프 네트워크 런타임 플로우 및 방어 기제

### 3.1 알림 전파 아키텍처 동작도

```text
[호스트 물리 서버 런타임 재시작 타점]
   ↓
SlackNotifier 파이썬 싱글턴 초기화 (.env 파싱 리딩)
   ├─ startup_grace_sec 셧업 유예 타이머 가동 (디폴트 15분 세그먼트)
   └─ Circuit Breaker 상태: 대기 수신 (CLOSED)

[최초 15분 타이머 잔여 이내 구간]  ─── 일체의 모든 타전 알림 100% 강제 억제 뮤트 (콜드 스타트 기동 보호)
[15분 타이머 돌파 이후 정상 궤도]
   ↓
LogCollector 수집 덤프 인입 → check_and_alert(logs) 미들웨어 심사
   ├─ _should_alert() 자격 검증망 통과 여부 심사
   │    ├─ [지정된 활성 업무시간대 영역인가?] → 위반(심야 등) 시 패스
   │    ├─ [차단 배제된 블랙리스트 서비스 출신인가?] → 적발 시 패스
   │    ├─ [위험 레벨 기준치 미달인가?] → 적발 시 패스
   │    └─ [지정된 노이즈 필터링 텍스트 문구인가?] → 발각 시 영구 패스 보류
   │
   ├─ notify_critical_error() 최종 트리거 타격 명령
   │    └─ send_message() 외부 훅 핑 교신
   │         ├─ 체크 A: [내부 Circuit 차단기가 현재 OPEN(차단 중)인가?] → 스킵
   │         ├─ 체크 B: [Rate Limit 속도 한도를 현재 침범했는가?] → 포착 시 차단기를 강제 OPEN 스위칭하고, 관리자에게 무음 셧다운 개시 경고장 1건 타전 후 징벌적 락 다운 전환.
   │         └─ 최종 체크 패스 후 전송 완료 시 → Rate 누적 카운터 ++1 기록
   │
   └─ track_error() 관측 스파이크 모니터링 → spike_threshold 초과 돌파 → 병합 타전 교신 시도

AutoRecovery 봇 자동 복구 개입 종료 후 → notify_auto_recovery() 연계 → send_message() 타전
```

### 3.2 Circuit Breaker (도달 퓨즈망) 상태 머신 토폴로지

에러 스팸 연쇄 폭격으로 발생하는 워크스페이스 도배 디도싱 현상을 스마트하게 셧다운 차단 방어 수행. 

```text
정상 궤도 진행 (CLOSED) ──(어뷰징 한계 임계치 돌파 적발)──▶ 회로 파단 (OPEN 망 차단 진입) ──(가혹한 차단 지정 쿨타임 시간 경과 완료)──▶ 복구 테스팅 타진 (HALF_OPEN 반개통)
  ▲                                                                                                    │
  └──────────────(시운전 임시 테스트성 핑 메시지가 다시 정상 스루 전송 통과 성공 시)────────────────────────────────────┘
```

> **지배적 징벌 로직 룰**: 파단된 회로(OPEN)가 복구가 실패하고 연속해서 오버부하 타격으로 락다운 진입 시, **차벌적 쿨타임 지정 시간이 배수(2배 단위)로 기하급수적 누적 지수 폭증 전개**됨. (예: 30분 밴 → 연이은 실패 시 60분 밴 → 120분 밴 → MAX 상한 최대 8시간 밴 칩거 모드로 전환 통제 보완됨)

## 4. 커스텀 기동 노이즈 정규식 필터 블록 사전

서버 재런타임 시퀀스 부팅 직후 빈번히 일어나는 의례적인 대기 파편 텍스트를 탐색하여 알림 발신을 무조건 폐기 묵살함.

| 타겟 서비스 파이프라인 | 필터 억제 적용 정규 패턴 문자열 키워드 |
|--------|-----------|
| 하둡 생태계 (Hadoop) | `incompatible clusterids`, `initialization failed for block pool`, `namenode is in safe mode` |
| 카프카 브로커 (Kafka) | `nobrokersavailable`, `broker is not available`, `leader not available` |
| 스케줄러 (Airflow) | `scheduler is not running`, `dag file processor manager` |
| 시스템 공용 통신 | `connection refused`, `connection reset by peer`, `temporary failure in name resolution` |

## 5. 관리자 제어 대시보드 옵션 연계 안내 (UI 세팅)

관리자 패널 도메인 `/admin/logs` 페이지 접속 후 슬랙 전용 컨트롤 박스에서 실시간 조준 옵션 튜닝을 제공함. 저장 버튼 액션 트리거는 인메모리 프로세스에 즉시 열을 뿜으며 핫픽스 실적용되지만 지속 유지를 위해서는 필연적으로 물리 `vi .env` 수동 편집 수정 반영이 반드시 하달되어야 결함이 존재 방어됨.

| UI 표출 도면 | 통제 기능 상세 제어 속성 설명 |
|----|------|
| **Webhook 통신망** | 슬랙 채널 송신 URL 텍스트 박스 세팅 바인딩, 동시 통신 여부 체크 가도용 라이브 핑 수동 테스트 슈팅 슛 버튼 포함 |
| **기준 레벨 & 타임존 룰** | 발송 기준 미달 필터 하한 커트라인(CRITICAL 등) 및 한국시간 KST 업무 허용 시간 존 활성 타이머 박스 조절 |
| **초당 마비방어 스로틀링 캡** | 기동 유예 타임 설정 / 시간 내 최대 송파 건수 제한 / 차단 퓨즈 박스 기나긴 징벌 쿨타임 조정 인터페이스 제공 |
| **블랙리스트 제외 타겟** | 터져도 알람 타전을 무적 스킵 무시 배제할 고철 컨테이너 서비스명 블랙 박스 리스트 투입 단자 기입 제공 |
| **Circuit Breaker 현황판** | 알고리즘 잔여 대기 스팬 차징 현황 타이머 쿨, 카운터 랭크업 도식 테이블 게이지 등 라이브 시스템 모니터링 판 표시 적용 |

## 6. 트러블슈팅 및 엣지 케이스 필수 운용 수칙
1. **휘발성 세팅 지배 규칙**: 브라우저 UI에서 세이브(Save)하는 값은 임시 휘발성임을 망각하지 말고 핵심 변동 롤 모델 값은 롤백 파괴 방어를 위해 백그라운드 서버 내 `.env` 시스템 코어 본체 수정으로만 절대 해결 하달 지시 요망.
2. **KST 자정 타임 교차 한계 역행 변칙 연산 지원 체계**: 활성 타임존 로직에서 야간 무음 록 설정으로 심야 22시 오픈 ~ 익일 06시 오프 밴 등 자정 00시를 역교차 이관하는 구간 박스 입력 지정 시에도 내부 백엔드 연산 함수 모델 모델 변환 캡처 알고리즘으로 에러 유발 없이 100% 능히 연산 소화 지원 스루.
3. **`min_alert_level` 연동 상쇄 체계의 치명성 파이프라인 구속력**: 10분에 15번 이상 스파이크가 폭증할 때 알려주는 급증 탐지기 또한, 마스터 등급인 최소 알림 필터 레벨(`min_alert_level`)의 권위 통제 필드 체계를 절대적으로 100% 추종 의존 바인딩 종속함. 세팅을 `CRITICAL`로 묶어두면, 저 스파이크 폭증 누적 카운트를 세는 주머니 내부 스코어 보드 게이지에도 일반 찌끄레기 `ERROR` 노이즈는 일절 담기지도 카운트되지도 못하게 필터 배제 통과시켜 록 차단 방어함.
